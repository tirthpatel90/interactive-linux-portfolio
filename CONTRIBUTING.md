# Contributing to Interactive Linux Desktop Portfolio

First off, thank you for considering contributing to the Interactive Linux Desktop Portfolio! It is people like you who make this project so awesome.

This document outlines a set of guidelines for contributing to this repository.

---

## 🗺️ How Can I Contribute?

### 🐛 Reporting Bugs
Before creating bug reports, please check the [existing issues list](https://github.com/tirthpatel90/My-Portfolio-/issues) to make sure the issue hasn't already been reported.

When opening a bug report, please include as much detail as possible:
* **Title:** Use a clear and descriptive title.
* **Environment:** Include your OS, browser, and version.
* **Steps to Reproduce:** Provide step-by-step instructions.
* **Expected vs. Actual Behavior:** Explain what should have happened vs. what actually happened.
* **Screenshots:** If applicable, add visual examples to help explain your problem.

### 💡 Suggesting Enhancements / Features
If you have an idea to improve the portfolio:
1. Check the [existing issues list](https://github.com/tirthpatel90/My-Portfolio-/issues) to verify if the idea is already suggested.
2. Open a feature request explaining what capability you would like to see, why it is useful, and how it might be implemented.

### 🔧 Submitting Code Changes (Pull Requests)
We welcome code enhancements, styling polish, and documentation updates! To submit a change:

1. **Fork the Repository:** Create your own fork of the codebase.
2. **Clone Locally:** Clone your fork to your computer.
3. **Create a Feature Branch:** Choose a descriptive name for your branch.
   ```bash
   git checkout -b feature/cool-new-capability
   ```
4. **Make Your Changes:** Follow the project's styling and structure principles:
   * Keep Javascript vanilla (clean client-side ES6+).
   * Document new configuration keys in `config.js`.
   * Maintain the glassmorphism theme layout and responsive stylesheet rules.
5. **Commit Your Code:** Write clear, concise commit messages following standard conventions:
   ```bash
   git commit -m "feat: add cyber security scanner terminal emulator command"
   ```
6. **Open a Pull Request (PR):** Submit a PR to the upstream default branch. Fill out the PR template with all necessary context.

---

## 📜 Code of Conduct
Please be respectful and supportive to other contributors, maintainers, and issues participants in the community. Let's build cool developer portfolios together!
